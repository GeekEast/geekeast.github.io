Node.js App CI/CD Example with Elastic Beanstalk and Circle CI

2019-05-23 | 2019-12-12 | Developer , CICD , AWS |
Symbols count in article: 6.3k | Reading time ≈ 5 mins.

Simple Skeleton of CI/CD

Related Concepts:

  • CI: Continuous Integration: Merge Branch to master after passing the building and testing stage
  • CD:
    • Continuous Delivery
    • Continuous Deployment: run the code in production server(s)

Tools:

  • Node.js with yarn
  • Git
  • Github
  • Circle CI
  • Elastic Beanstalk

About Elastic Beanstalk

  • Configure the production environment for your code, for example with Ubuntu 16.04 and Node.js 10.
  • Configure the command to start runing your application, for example npm start.
  • Configure security, load balancing and auto scaling service from aws.

Steps from Scratch

Create your project Locally

    1. Install Node.js and Yarn on Ubuntu 18.04
    1. Create the project
    1. Copy and paste the code to app.js
    1. Run the app on local machine
  1. If app runs successfully, you will see “Hello World!” in localhost:3000

Set up GitHub

    1. Register for an account
    1. Create a repository the following.

    1. Push your code to github
    1. Github intializaion if completed.

Configure Circle CI

    1. Sign in Circle CI using Github Account
    1. Add Project on the left of page
    1. Select your project in github and click on Set Up Project
    1. Add the configuration file
    1. Copy and paste to config.yml
    1. Add .gitignore file
    1. Copy and paste to .gitignore file
    1. Push to Github
    1. Click Start Building on CircleCI Page
    1. Then circle ci will show it build and test your app successfully.
    1. After this, everytime you push code to your master of git repo, it will trigger cicleci to automatically build and test your code.

Configure Elastic Beanstalk (Locally -> AWS)

  • aws online
    • Sign up an aws account
    • create the elastic beanstalk full access role to the user
    • create the user credentials and downloaded to your local machine
  • aws locally

    • install and configure awscli

    • configure credentials

    • install awsebcli

    • initialize eb in your project

    • modify the .gitignore file

    • first deploy

Configure Elastic Beanstalk with Circle CI

  • Set AWS credentials as Environemnt Variable in Circle CI
  • Modify the ebs configure yaml file
  • Each time you merge a branch to master, to deployment to elastic beanstalk will be executed

Customize the start command in elastic beanstalk

  • modify the node-settings.config file
  • modify the package.json file

Important Notes

  • After 1st deployment from local, you could use aws ebs console to create environment and customize more configurations like security group, auto scaling and load balancing
  • If there is any error, please use the aws eb console > logs to inspect the issue.

Not Covered:
